# AI 漫剧产品分析与改进建议

日期：2026-05-07

## 结论

本项目继续选择 `LumenX Studio` 做主基座，`LocalMiniDrama` 做桌面轻量备选。`huobao-drama`、`Toonflow-app`、`BigBanana-AI-Director` 都值得持续跟踪，但不适合作为当前 2-3 人试制团队的第一基座。

核心判断不是 star 最大，而是先把“低成本、可二开、可跑通、可控角色一致性、可逐步接新模型”的生产闭环稳定下来。

## 为什么不使用指定项目作为主基座

### superpowers-zh

- 定位是 AI 编程 skills 方法论，不是 AI 漫剧生产系统。
- 它适合提升团队用 Codex/Claude/Cursor 做工程协作的效率，可以借鉴为团队工作规范。
- 它不能替代剧本解析、角色资产、分镜、图生视频、配音、合成这些产品链路，所以不作为漫剧基座。

### Toonflow-app

- 产品定位很接近：开源一站式 AI 短剧创作工具，覆盖 AI 编剧、分镜、角色与视频生成。
- 优点是热度高、桌面端路线清晰、Agent/技能/本地模型/对象存储目录规划完整。
- 当前不选它做第一基座的原因是复杂度更高：Agent 体系、桌面打包、内置 Web、运行时数据、技能、模型管理等模块会拉高学习和维护成本。
- 对小团队而言，早期更需要快速试题材、控废片率、算单镜头成本，而不是先吃下一个重型桌面创作系统。
- 建议：把它作为“普通运营同事桌面工具”和“Agent 生产组织方式”的参考，等 LumenX 跑通样片后再二次评估。

### BigBanana-AI-Director

- 产品理念很强，特别是 `Script-to-Asset-to-Keyframe`、项目/季/集组织、世界观资源、首尾帧和关键帧控制。
- 但许可证明确限制商业使用，需要另行商业授权；这与“通过 AI 漫剧赚钱”的目标有直接冲突。
- 仓库更适合作为工业化流程蓝图参考，不适合作为可控二开基座。
- 建议：借鉴它的“世界观资产库、关键帧、提示词版本回滚、镜头预审”方法，不直接依赖其代码或服务。

### huobao-drama

- 热度最高，TypeScript 全栈平台感强，覆盖剧本、角色、分镜、视频合成。
- 风险点是平台化较重，并且仓库根目录页面未展示清晰的 `LICENSE` 文件；README 中还导向商业版，商业二开边界需要进一步法务确认。
- 对 2-3 人内容试制团队来说，先接入完整平台、用户体系、Agent 技能和数据库栈，容易把精力从“内容验证”转移到“平台维护”。
- 建议：观察它的短剧产品信息架构和商业化包装，但当前不作为主基座。

## 为什么继续选 LumenX

- 源码完整，MIT 许可，适合商业试制和二开。
- 工作流正好覆盖 AI 漫剧关键链路：资产提取、风格定调、资产生成、分镜图、分镜视频、拼接成片。
- FastAPI + Next.js 结构清晰，当前项目已经基于它搭建，继续推进比换基座成本低。
- 天然接入 Qwen/Wanx，适合用 DashScope-only 先跑低成本样片。
- 已有角色资产、三视图、头像、参考图、i2v/r2v、Batch Size 抽卡和最终候选选择的产品形态，和漫剧生产痛点吻合。
- 后续可以通过 provider registry 保留 Kling、Vidu、PixVerse、Seedance 等模型扩展空间。

## 产品改进优先级

### P0 打开即稳定

- Docker 前端所有浏览器 API 请求都必须正确代理到后端。
- 前端对集合接口返回值做类型保护，避免 HTML 或错误对象被当数组渲染。
- 首页空项目、空系列、后端不可用、密钥未配置都要有可理解的错误提示。

### P1 镜头级抽卡台账

- 每个镜头记录：模型、prompt、参考图/参考视频、seed、分辨率、时长、费用、生成耗时、人工评分、采用原因。
- 目标是把“抽卡师的辛苦”转为可复盘数据：哪些题材、角色、动作、模型组合最容易出片。
- 后续可以基于台账做自动重试、自动淘汰低分候选、预算预警。

### P1 剧集级一致性资产库

- 把角色、场景、道具、服装、世界观、禁用设定沉淀到系列级别。
- 单集只引用和继承，减少每集重新提取导致的人设漂移。
- 对角色增加“可出镜服装包”和“不可改变项”，例如脸型、发型、瞳色、标志性道具。

### P1 模型选择与成本控制

- 默认低成本模型跑全链路，关键镜头才升级到更贵的视频模型。
- 模型配置要按任务类型分层：编剧/分镜润色、文生图、图生图、图生视频、参考视频驱动、TTS。
- 每次生成后写入成本估算，形成单集成本和单镜头成本。

### P2 团队协作与交付

- 先不做自有短剧 App、支付、会员、积分。
- 先做内部生产工具：题材试制、样片评估、B2B 试播集交付。
- 小团队三人以内的分工建议：技术/流程、导演剪辑/审片、题材运营/发布数据。

## 可赚钱赛道建议

- 第一阶段：做 3 个 60-90 秒竖屏样片，用抖音、快手、B 站、小红书、YouTube Shorts 验证完播和互动。
- 第二阶段：胜出题材扩成 5 集短系列，沉淀可复用角色和世界观资产。
- 第三阶段：面向网文作者、MCN、小版权方做试播集或低成本样片服务，按条收费或联合分成。
- 第四阶段：积累镜头台账和稳定 SOP 后，再考虑工具化对外服务。

## 技术解决抽卡师辛苦的方向

- 把抽卡从“盲生成”变成“镜头实验”：每次只改变一个变量，例如参考图、运动提示、模型或 seed。
- 把复杂动作拆成多个 3-5 秒短镜头，降低单次生成失败概率。
- 用首尾帧/关键帧约束动作起止，减少角色乱动和镜头漂移。
- 用台账记录失败原因，后续自动避开高废片提示词和高废片模型组合。
- 用低清/短时长预检，过关后再升清晰度或更贵模型。
